Las sentencias más usadas de Git

Git es un sistema de control de versiones distribuido de código abierto diseñado para ser rápido y eficiente

Git es una potente herramienta que no solo nos permite trabajar de forma grupal en los proyectos, sino que además nos permite recuperar nuestro trabajo de errores, volviendo a estados estables anteriores.

Por ello, en esta ocación vamos a dar un repaso a las 4 sentencias más usadas en Git

  • Git Status

    Si bien Git no tiene una interfaz gráfica y se ejecuta en la consola, el comando Git Status nos brinda en pantalla toda la información relevante para estar al tanto del estado de nuestro repositorio

      Entre la información que se desplega en pantalla se incluye:
    • Rama o Branch en la que estamos trabajando
    • Archivos nuevos/borrados/modificados o sin trackear en nuestro proyecto.

  • Git Add

    El comando Git Add nos permite realizar un seguimiento a nuestros archivos, de manera que los abarcamos para guardar sus cambios al realizar el siguente Commit.

    El comando Git Add, incluye comodos atajos tales como:

    • Git add *

      Nos permite añadir a todos los archivos del directorio y subdirectorios. Y con todos... nos referimos a TODOS, ignorando los abarcados por el .gitignore

    • Git add *.extension

      Nos permite añadir todos los archivos de la extensión dada, de este modo filtramos la lista a añadir evitando abarcar archivos no deseados en el Commit.

  • Git Commit

    En pareja con el 'Git Add' el comando 'Git Commit' es muy utilizado, nos permite guardar cambios como puntos a los que podremos volver cuando lo consideremos necesarios, ya sea por un error crítico o por haber ensuciado el código probando una idea que queremos ya sea descartar o comenzar a implementarla desde 0.

  • Git Push

    Lo grandioso de Git para por compartir nuestro trabajo ya sea en privado o abierto al público.

    El comando Git Push 'nombre_del_enlace_git' 'nombre_de_la_brach' publica los commits presentes en la rama indicada al enlace del repositorio git online. De esta forma se compartimos e integramos nuestros avances al repositorio online que tengamos con nuestros compañeros colaboradores.
